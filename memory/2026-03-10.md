## 邮箱验证码登录 + 用户隔离 (commit 6d894f0)

### 新增文件
- `src/lib/auth.ts` — Auth.js v5 配置（JWT + Credentials email-otp）
- `src/lib/auth-api.ts` — requireUserId() + 统一 401 处理
- `src/lib/email.ts` — Resend 验证码邮件
- `src/app/api/auth/[...nextauth]/route.ts` — Auth.js handler
- `src/app/api/auth/request-code/route.ts` — 验证码发送（60s限频、SHA-256哈希、5分钟过期）
- `src/app/login/page.tsx` — 邮箱+验证码登录页
- `src/app/(protected)/layout.tsx` — 服务端鉴权保护
- `proxy.ts` — Next.js 16 路由代理保护
- `src/types/next-auth.d.ts` — 类型扩展

### Schema 改动
- 新增 `users` 表（id, email, display_name, status, created_at, updated_at, last_login_at）
- 新增 `auth_codes` 表（id, email, code_hash, expires_at, consumed_at, attempts, created_at）
- `tasks` 表新增 `user_id` 字段
- DB 连接改为异步 `getDb()`，自动选择 SQLite(本地) 或 Turso(生产)

### API 用户隔离
- tasks: 所有 CRUD 加 user_id 过滤
- report: 查询任务限定 user_id
- extract/asr: 登录校验

### 页面迁移
- 主页/任务/报告移入 `(protected)` 路由组
- BottomNav 新增退出登录按钮
