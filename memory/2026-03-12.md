# 开发日志 2026-03-12

## Vercel 时区修复 (commit 614ca63)

### 问题
- Vercel serverless 默认运行在 UTC 时区，导致 `dayjs()` 获取的"今天"、"当前时间"与东八区不一致
- Vercel 将 `TZ` 列为保留环境变量，无法通过 Dashboard 设置

### 方案评估
| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| Vercel 设 `TZ=Asia/Shanghai` | 零代码改动 | `TZ` 是保留变量，不允许设置 | 不可行 |
| 代码层 dayjs wrapper | 显式控制，不依赖部署环境 | 需改所有 import | 采用 |

### 实现
1. 新增 `src/lib/dayjs.ts` — 统一 dayjs 配置
   - 加载 utc + timezone 插件
   - `dayjs.tz.setDefault("Asia/Shanghai")`
   - wrapper 函数：`dayjs(...args).tz()` 确保每次调用都转东八区
   - `Object.assign` 保留 dayjs 静态方法
2. 7 个文件 import 替换：`import dayjs from "dayjs"` → `import dayjs from "@/lib/dayjs"`
   - `src/app/api/tasks/route.ts`
   - `src/app/api/ai/extract/route.ts`
   - `src/app/api/ai/report/route.ts`
   - `src/lib/due.ts`
   - `src/hooks/useReminders.ts`
   - `src/app/(protected)/page.tsx`
   - `src/app/(protected)/reports/page.tsx`

### 关键细节
- `dayjs.tz.setDefault()` 只对 `dayjs.tz()` 生效，不影响 `dayjs()`，所以需要 wrapper
- 客户端（浏览器）本身就用用户本地时区，此修复主要解决服务端问题
- `isoWeek` 插件在 `report/route.ts` 中单独 extend，无需调整

### 构建验证
- `npx next build` 通过，无类型错误
